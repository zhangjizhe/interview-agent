② 为什么能省 79,131 tokens · 4 大来源
1Provider 自动降级 · 永久错 disable ~30%
401 / 402 / 403 / 404 永久错自动 disable,5xx / 429 走 fallback。 DeepSeek 充值前 33.3% fallback,复活后 0% — 健康路径占满, 50 轮 0 次意外 fallback,LLM 调用稳定收敛。
2Context 复用 + 4 级水位线压缩 ~30%
LangGraph PostgresSaver checkpoint 维护对话历史,system prompt 跨轮复用; context 满 60% / 80% / 95% 三段触发 Snip / Prune / 摘要, 避免长对话 token 线性增长。
3Cache 工程兜底 (Prefix / Semantic / Exact) ~30%
3 段 Prompt Prefix 识别 (SYSTEM / SEMI_STATIC / DYNAMIC) + Semantic 黑白名单 + Redis 精确层; 当前 provider 限制下命中率 0%,但 代码完整可启用 (切 Anthropic Claude 即可达理论 ≥65%)。
4Cost Panel 实时反查 · 避免重算 ~10%
Redis HINCRBY 实时 counter + 5 次刷盘防抖, 7ms 响应 (目标 1000ms) 让前端 dashboard / 监控 / 计费 0 延迟; 避免每个 session 重新聚合 token,减少 LLM 调用。
