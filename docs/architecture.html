05 面试深挖示例

真实会被问的问题 + 我的标准回答方向。

DeepAgents vs LangGraph？

FOLLOW-UP

DeepAgents 内置 SubAgent + Plan-and-Execute，开箱即用； LangGraph 更底层，要自己画状态机。我选 DeepAgents 是因为 SubAgent 抽象正好对应"面试官 / 评估官 / 总结官"分工，不需要从零画图。

为什么需要 3 层记忆？

FOLLOW-UP

单一存储不划算：Redis 实时但丢；Mem0 语义但慢；Milvus 简历 RAG 维度不一样（候选人画像 vs 简历原文）。三层分别优化，互不污染。

stub 滑窗为什么是 Cache 杀手？

DEEP

Prompt Cache 按 prefix 字节匹配，stub 决策一旦来回滑，全段 prefix 字节变 → 缓存全失效。必须单调推进才能保住 Cache 命中。

Mem0 Cloud vs 自托管？

OPS

Cloud 走 SaaS（Token 鉴权即用，0 部署成本）；OSS 自托管需要 Postgres + pgvector + Python 服务 = 2 个容器。简历项目用 Cloud 更聚焦业务逻辑，商用切 OSS 拿数据主权。
